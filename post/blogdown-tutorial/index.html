Blogdown Tutorial

Last updated on Sep 13, 2019 2 min read

Install R and RStudio

R: https://cloud.r-project.org/

R Studio https://www.rstudio.com/products/rstudio/download/

Install blogdown package

install.packages(“blogdown”)

Install hugo

blogdown::install_hugo()

Create a Personal Static Website with Academic Template

Create a folder on you computer, and set the working directory to the new folder.

For example:

Windows: setwd(“C:\folder”), remeber use double backslash between different folders.

Mac: setwd(“/User/name/desktop/folder”)

blogdown::new_site(theme = “gcushen/hugo-academic”)

  • dir is the directory that you want your store your website

  • the content in theme contains github username of the theme’s author and the repository name of the theme. They are separated by backslash(/).

Hugo Themes: https://themes.gohugo.io

Access Your Website

If you want to access your website, go to RStudio: 1. set working directory to your website folder 2. run “blogdown::serve_site()” 3. type “http://127.0.0.1:4321" in your browser to access your site.

Using Git on Windows

Download Git Here: https://git-scm.com/download/win

Config Git:

git config –global user.email “youremail@host.com”

git config –global user.name “Your Name”

Using Git on Mac

No need to install Git. Just use “Terminal”.

Push your website to your github

  1. start git bash (on Windows), or Terminal (on Mac)

  2. In your command line, Use cd directory to go to the public folder of your website. Set your current working directory to the folder which contains your website.

  3. git command:

  • git init

  • git add .

  • git commit -m “what you commit”

  • git remote add origin https://github.com/your_github_name/repository_name.git

  • git push -u origin master

Publish Your Website

repository -> Settings -> GitHub Pages

Modify Your Website Adapting to Your Needs

  • config.toml
  • content => home
  • content => author
  • Learn more: https://bookdown.org/yihui/blogdown/

Problem Sovling

  1. If your published website does not look like what you see on your computer,

go to config.toml file, and add “relativeurls=true“ under the line “baseurl=”/“”.

  1. Cannot call hugo:

a. options(blogdown.hugo.dir = ‘custom directory’)

b. blogdown::install_hugo(), the hugo will be installed in an custom directory.

  1. How can I make changes in the future?
  • Use RStudio to serve your site. a. Open RStudio set the working directory to your website folder.

b. In “Console”， type “blogdown::serve_site()”

c. Make changes in the .md files, and save

d. See your changes in your broswer.

After you have done the changes:

  • Push the changes to GitHub

a. Open git bash on windows, or Terminal on Mac

b. Change your directory to the public folder using cd command.

c. Use git commands to push your changes

 \-  git add . (add all the changes to local repository, the dot(.) means "all")

 \-  git commit -m "what you have done"  ("what you have done" - any changes you made)

  \-  git push
